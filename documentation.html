1.Introduction

1.1 Purpose

The product described in this documentation represents the frontend part of a bigger project named AuCo(Autograph Collector).
This SRS encapsulates version 1.0 of this product, which is composed of HTML, CSS and JavaScript files.

1.2 Document Conventions

This document is written using Times New Roman. Priorities are indicated by the <h1> - <h6> tags, with the highest one represented by the <h1> tag.
The content is structured in different sections using the "Scholarly HTML" format. Each section has its own number in accordance with the priorities mentioned above.

1.3 Intended Audience and Reading Suggestions

The documentation addresses the project manager and the developers involved. It describes the product, the requirements and the implemented features. The most pertinent sections for the project manager are 1.4 Product Scope and 2.3 User Classes And Characteristics. The sections regarding code implementation, and the most relevant for the developers, are 2.2 Product Functions, 2.5 Design And Implementation Constraints and 4 System Features.

1.4 Product Scope

AuCo is a useful tool for amateur autograph collectors, that allows users to organize their acquired autographs and store their unique specifications and descriptions. This website helps collectors access their own autograph database easily and gain insight on some global statistics. AuCo uses a personal system of estimating values within the app, and every autograph has a value expressed in auco points. Also, users can access their own statistics in order to see information like their total value acquired in points.

2 Overall Description

2.1 Product Perspective

This product is part of the Collection-Organizing-Apps-family, and gives the user an overview of their own collection of autographs in the "Statistics" section.

2.2 Product Functions

  1. User Interface:
    1. Creating an account
    2. Adding an autograph
    3. Editing the profile
    4. Searching through the autographs
    5. Changing settings
    6. Account personalization
    7. Seeing personal and global statistics
  2. AuCo system
    1. Database storing system

2.3 User Classes and Characteristics

This product is destined to all classes of users. The autograph collection community has no age limit. The main characteristic of the users is having a valid email address. Also, the average user will show enthusiasm regarding collecting autographs and storing them. Even a person with just one autograph can find AuCo really helpful. Every AuCo functionality is accessible for all the users.

2.4 Operating Environment

The product can optimally operate in any web browser that supports HTML5.

2.5 Design and Implementation Constraints

The main constraint that AuCo imposes on the user is a superior limit in reference to the possible number of autographs stored. A single profile can store up to 2000 entries.

2.6 User Documentation

The AuCo website comes with self-explanatory functionalities, and all of them are indicated by specific buttons with suggestive names like "Log out", "See your stats" or "New autograph".

2.7 Assumptions and Dependencies

This product has two dependencies. The main one is represented by the chosen hosting server and the other one consists of the database used for storing all the user data. Any problem that occurs regarding the database or the host provider that threatens this product can be solved by switching to another third party provider. In addition, any memory issues that may occur will determine AuCo to expand the available storage with the help of future investments.
